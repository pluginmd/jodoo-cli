# 03 — Tool Catalog (Shortcut → MCP Tool Mapping)

The registry is generated, not hand-written. This page documents the **mapping
rules** and the **current catalog** so reviewers can eyeball consistency.

## Naming rules

| Source | Transform | Example |
|--------|-----------|---------|
| `Shortcut.Service` | lowercase, used as namespace | `jodoo` |
| `Shortcut.Command` | strip leading `+` | `data-list` |
| Tool name | `{service}.{command}` | `jodoo.data-list` |
| Flag name | kebab-case → snake_case for MCP args | `--app-id` → `app_id` |

Why `.` instead of `/` as the namespace separator: MCP tool names are
typically matched loosely by models and dot syntax is friendlier for LLM
output ("jodoo.data-list" reads naturally).

Why snake_case for arguments: MCP SDKs and most shipped servers (`filesystem`,
`postgres`, `slack`) use snake_case. Claude produces snake_case JSON keys more
reliably than kebab-case.

## Synthetic arguments added to every tool

| Arg | Type | Maps to | When available |
|-----|------|---------|----------------|
| `dry_run` | boolean | `--dry-run` | always |
| `confirm` | boolean | `--yes` | only on `Risk == "high-risk-write"` |
| `format`  | string  | `--format` | only on `HasFormat == true`; defaults to `json` |
| `jq`      | string  | `--jq` | always (advanced users) |

## Risk gate

| Shortcut risk | MCP behaviour |
|---------------|---------------|
| `"" / "read"` | Always callable |
| `"write"` | Callable; CLI validates normally |
| `"high-risk-write"` | **Requires `confirm: true`**; otherwise returns an error content block |
| `--read-only` mode (server flag) | Only `read` shortcuts are exposed; the rest don't appear in `tools/list` |

## Example: auto-generated schema

From `appList` (`shortcuts/jodoo/app.go:17`):

```go
var appList = common.Shortcut{
    Command:   "+app-list",
    Risk:      "read",
    HasFormat: true,
    Flags: []common.Flag{common.LimitFlag(100), common.SkipFlag()},
}
```
